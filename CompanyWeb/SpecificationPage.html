Welcome Page About Page Specifications Project Plan Our Team Page

As lovers of indie content, it can be challenging to find all the different resources available to promote movies, tv shows, music and games. Not to mention many of the platforms are single topic based so we would have to keep track of each site for each category, so my team and I wanted to create a universal platform that consolidates all categories of indie content together where consumers can filter through any type of media they please without having to run back and forth between all different types of places.


Purpose of project

By creating our database, users are easily able to find new content suggestions each week and are asked to review them if consumed. This helps both people find exactly what they didn't know they were looking for as well as giving the creator's real time feedback on their work.


List of deliverables

Creative Brief:
1. Project Scope
NextWave Collections is an indie content repository where users can come and seek out new content to consume that may be lesser known to the public as well as review the same content.

2. Target Audience
Anyone is welcome to use the platform, however we are mainly targeting those who seek out content that may not be as main stream to the public.

3. Overall Message
We want to help the world understand that great media can come in all shapes and forms, and just because content is not number one at the box office or top of the trending charts does not mean it is low quality. We generally want to help generate awareness to the diamonds in the rough.

Database Design

Database layout for Indie Website:
Table: dbo.tblUsers
Fields and data types: UserID(int), FirstName(varchar), LastName(varchar), Email(nvarchar), Username(nvarchar), Password(nvarchar)

Table: dbo.tblMovies
Fields and data types: MovieID(int), MovieName(nvarchar), CreatorID(int), Review(varchar), Rating(int)

Table: dbo.tblBooks
Fields and data types: BookID(int), BookName(nvarchar), CreatorID(int),
Review(varchar), Rating(int)

Table: dbo.tblSongs
Fields and data types: SongID(int), SongName(nvarchar), CreatorID(int),
Review(varchar), Rating(int)

Table: dbo.tblTVShows
Fields and data types: ShowID(int), ShowName(nvarchar), CreatorID(int),
Review(varchar), Rating(int)

Table: dbo.tblCreators
Fields and data types: CreatorID(int), FirstName(varchar), LastName(varchar), TypeID(int)

Table: dbo.tblMediaType
Fields and data types: TypeID(int), Type(varchar), CreatorID(int)


Explanation: The Users table will hold all of the standard user information when a user signs up for the site. Each type of media table will contain identifying fields, along with being connected to the Creators table to identify who created it. This will promote normalization. There will be a stand-alone Media Type table to hold the different types of media being uniquely identified with a TypeID field that the Creators table can call from.
